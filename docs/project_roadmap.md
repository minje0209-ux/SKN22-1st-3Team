# 국내 자동차 시장 분석 프로젝트 – 확장 로드맵

본 문서는 **현재 스키마 및 DDL과 충돌하지 않는 형태로 정리된 ‘확장 기능 로드맵’**이며,  
MVP(현 버전) 이후 고려할 확장 기능들을 명확히 기술한다.

현재 프로젝트의 핵심 데이터 소스는 다음 네 가지로 유지된다:

- 다나와 판매실적 / 인기순
- 네이버 데이터랩 검색량
- 네이버 블로그 상위 3개 + 워드클라우드
- 공공데이터(등록 현황)

이 문서의 항목들은 **향후 개발 시점에 선택적으로 도입 가능한 확장 기능들**이다.

---

## 1. 관심도(Interest) 데이터 확장

### 1-1. Google Trends 지표 통합 (보류)

- 현재 DDL에 `google_trend_index` 컬럼만 존재하며, NULL 허용.
- MVP에서는 사용하지 않음.
- 향후 도입 시 작업:
  - pytrends 기반 월별 지표 수집
  - 모델명 매핑(검색어 정규화)
  - 상대지수(0~100)를 기존 관심도와 정규화해 결합
- 목적:
  - 국내(네이버) + 자동차 플랫폼(다나와) + 글로벌(구글) 3축 관심도 구성

---

## 2. 영상 기반 관심도 확장

### 2-1. YouTube Data API 활용

- 모델명 기반 월별 영상 데이터 수집:
  - 신규 업로드 영상 수
  - 상위 N개 조회수 합계
  - 좋아요/댓글 등 상호작용 지표
- 확장 테이블 예시:
  - `youtube_model_monthly`
- 도입 효과:
  - 리뷰/시승 중심의 실사용자 관심도를 정량화
- 고려 사항:
  - Google API 쿼터 관리
  - 국가별 시청자 구분 어려움

---

## 3. 텍스트 데이터 확장

### 3-1. 블로그 외 커뮤니티 텍스트 수집

- 후보:
  - 네이버 카페
  - 보배드림
  - 클리앙 자동차 포럼
- 목적:
  - 실사용자 경험 기반의 텍스트 데이터 확보
- 고려사항:
  - 비공식 크롤링 시 차단 리스크
  - 데이터 정규화와 라벨링 필요

### 3-2. AI 기반 텍스트 요약 및 감성 분석

- 블로그·카페 텍스트 자동 요약
- 긍정/부정 감성 평가
- 주제(가격/연비/디자인 등)별 분류
- 사용 모델:
  - KoBERT, KR-SBERT, GPT API 등

---

## 4. 공공데이터 활용 심화

### 4-1. 모델-세그먼트 자동 매핑 알고리즘

- 공공데이터는 개별 모델명이 없고 차종·연료 수준만 제공.
- 해결 방향:
  - 스펙 기반(전장/전폭/배기량/차체 형식)으로 세그먼트 자동 추론
- 목적:
  - 모델별 시장 점유율 흐름을 정교하게 결합

### 4-2. 지역/연료별 등록 현황 심층 분석

- 확장 기능:
  - 지역별 점유율
  - 연료별(전기·하이브리드 등) 성장률 분석
- 목적:
  - 특정 모델이 속한 시장의 구조적 변화 파악

---

## 5. 인프라/데이터 엔지니어링 확장

### 5-1. Raw 데이터 레이크 도입 (S3/MinIO + Parquet)

- 목적:
  - 스키마 변경 없이 장기 RAW 데이터 축적
  - 대용량 분석 대비

### 5-2. ETL 스케줄링 고도화

- 후보:
  - Apache Airflow
  - Prefect
  - Dagster
- MVP에서는 cron + Python 스크립트로 충분.

---

## 6. 시각화 및 알고리즘 확장

### 6-1. 경쟁 모델 자동 추천

- 기준:
  - 스펙 유사도 (피처 벡터)
  - 가격 거리
  - 관심도 패턴(네이버·다나와)의 유사도
- 기술:
  - Cosine similarity
  - 군집화(Clustering)

### 6-2. 시장 트렌드 예측 기능

- 예:
  - 유가 변화 → 하이브리드/전기차 관심도 예측
  - 특정 모델 단종/신규 출시 → segment 전체 변화 예측
- 기술:
  - Prophet / ARIMA / LSTM 기반 예측 모델
