# 네이버 블로그 기반 워드 클라우드 기능 설계 문서

## 1. 기능 목적

특정 차량 모델에 대해 네이버에서 검색했을 때 상위 노출되는 블로그 글 3개를 수집하고, 해당 글의 본문을 분석하여 자주 등장하는 단어를 기반으로 워드 클라우드를 생성하는 기능을 제공한다.
이 기능은 모델별 시장 반응 및 사용자 인식을 보조적으로 분석하는 데 목적이 있다.

---

## 2. 기능 개요

1. 모델명을 기반으로 네이버 블로그 검색 API를 호출한다.
2. 상위 노출 블로그 글 3개의 메타 정보(제목, 링크, 요약)를 저장한다.
3. 블로그 본문 HTML을 가져와 텍스트를 정제한다.
4. 형태소 분석기로 명사 기반 토큰화를 수행한다.
5. 단어 빈도를 계산하고 상위 키워드로 워드 클라우드를 생성한다.
6. 모델 상세 화면에서 블로그 요약과 워드 클라우드를 함께 보여준다.

---

## 3. 데이터 흐름

```
[모델명] → 네이버 블로그 검색 API → 상위 3개 결과 수집
          → 본문 HTML 크롤링 → 텍스트 정제
          → 형태소 분석 → 단어 빈도 계산 → 워드 클라우드 생성
          → Streamlit UI 렌더링
```

---

## 4. 사용 데이터 소스

* 네이버 검색 API (블로그 검색)
* 블로그 실제 본문 HTML (크롤링 필요)

---

## 5. 저장 정보

### 5-1. 블로그 메타 정보

* 제목
* 요약
* 링크
* 작성일(선택)
* 검색 키워드(= 모델명)
* 수집일자

### 5-2. 분석 데이터

* 본문 텍스트 원본
* 형태소 분석 결과(명사 리스트)
* 단어 빈도 Top N
* 생성된 워드 클라우드 이미지 경로

---

## 6. 정제 및 분석 과정

### 6-1. 텍스트 전처리

* HTML 태그 제거
* 광고 문구 제거
* 특수문자·이모티콘 제거
* 과도하게 반복되는 패턴 제거

### 6-2. 형태소 분석

한국어 형태소 분석기 선택지:

* khaiii
* mecab-ko
* okt

권장: **mecab-ko 기반 명사 추출**

### 6-3. 불용어 처리

* 자동차와 무관한 불용어 제거
* 의미 없는 단어 제외: 예) "이", "그", "저", "정말" 등

---

## 7. 워드 클라우드 생성

* 단어 빈도 Top N 기준
* 이미지 크기 800×600 정도
* Streamlit에서 PNG로 렌더링
* 생성된 이미지 파일은 캐싱하여 API 호출 부담 감소

---

## 8. UI 구성 (Streamlit)

### 모델 상세 화면에 표시할 요소

1. 블로그 3개의 제목 / 요약 / 링크
2. 워드 클라우드 이미지
3. 분석된 상위 키워드 목록(선택)

---

## 9. 캐싱 전략

* 블로그 검색 및 본문 분석은 하루 1회 갱신
* 동일 모델에 대한 중복 호출 방지
* 워드 클라우드 이미지도 동일 주기로 갱신

---

## 10. 향후 확장 가능성

* 네이버 카페, 지식인 검색 결과 추가
* 블로그 글 감성 분석(긍정/부정 스코어)
* 특정 키워드 기반 사용자 니즈 분석
* 리뷰 기반 객관적 평가 점수 생성

---

## 11. 결론

이 기능은 차량 모델별로 실제 사용자들이 어떤 주제를 많이 언급하는지 파악하고, 관심도 지표를 보조하는 데 유용하다. 검색량·판매량과 함께 사용자 서술 데이터까지 결합하면 시장 반응에 대한 다각적 분석이 가능해진다.
